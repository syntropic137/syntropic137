# Module Dependency Topology

This visualization renders every module in the monorepo (≥ 100 lines of code) as an interactive force-directed graph. Data is sourced directly from `.topology/graphs/dependencies.json` and `.topology/metrics/modules.json`.

- **Node size** reflects lines of code
- **Node color** indicates the bounded context
- **Edge opacity** scales with coupling weight
- **Scroll** to zoom, **drag** to pan, **hover** for module details

<TopologyDependencyGraph
  dependencies={depData.edges}
  modules={modData.modules}
  height={600}
/>

## Reading the Graph

Tightly-coupled modules cluster together due to the force simulation — if two modules have many dependency edges between them, they'll appear close together. Isolated modules drift to the periphery.

The color mapping groups modules by bounded context:

| Color | Context |
|-------|---------|
| 🔵 Blue | Orchestration / Workflow / Workspace |
| 🔷 Teal | Session / Observability |
| 🟣 Purple | GitHub integration |
| 🟢 Green | Artifact management |
| 🟠 Orange | Agentic Primitives |
| 🔵 Light blue | Event Sourcing Platform |
| 🟡 Yellow | Cost / Token tracking |
| ⚫ Gray | Other |

## Data Sources

The topology data is generated by the `syn-topology` analyzer and committed to `.topology/`:

- **`dependencies.json`** — 3000+ import edges between modules
- **`modules.json`** — per-module metrics (LOC, cyclomatic complexity, Martin metrics)
- **`coupling-matrix.json`** — pairwise coupling scores
